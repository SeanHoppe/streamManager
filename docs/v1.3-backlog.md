## From v1.2 close-out cycle (M1–M5, post-tag, 2026-05-03)

New findings surfaced during the soak finalization that postdate
`docs/v1.2-followup.md`. Tracked here so v1.3 has a single backlog
view. Originating cycle plan: `docs/v1.2-soak-finalize.md`.

- **`tools/cassette_record.py` filename collision** (M2, PR #49)
  Recorder derives filename from `_dt.date.today().isoformat()` only.
  Same-day re-record overwrites the prior cassette in place. M2
  recovered the prior cassette from the `v1.2.0` tag and suffixed the
  new one `-v12`, but a recorder hardening pass should add either a
  timestamp suffix or a `--no-clobber` guard.

- **Driver report does not separate ALLOW p95 from triggers** (M3, PR #50)
  `tools/soak_driver.py` aggregates routine ALLOW (n=50) with L2/L3
  (n=5) and L4 (n=5) into a single overall p95. ADR-5 budgets are
  per-band — the report should emit per-band p50/p95 so v1.3 ship-gate
  runs can compare to the budget table directly without manual
  reconstruction.

- **Driver report does not enumerate `LifecycleBridge._seen` final state** (M3, PR #50)
  Lifecycle bridge orphan-key check (Task C invariant) is not surfaced
  by the driver. Dashboard log shows no errors but final state is not
  positively asserted. Add a final-state dump to the soak report
  (acceptance gate per `docs/v1.2-soak-finalize.md` §M3).

- **REQUIREMENTS.md FR-OG-X session-picker entry never landed** (M4, PR #51)
  `docs/v1.2-task-plan.md` §B referenced adding an `FR-OG-X` entry
  for the session picker but `REQUIREMENTS.md` was not touched in the
  v1.2 cycle (last touched at `b66c840`, pre-v1.2). Spec-vs-impl drift.
  M4 explicitly deferred this rather than bundling. v1.3 task should
  audit Tasks B/C/D/E for any other FR-* drift and bring REQUIREMENTS
  spec version pin current.
