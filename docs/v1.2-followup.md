# v1.2 follow-up findings

Tracks review findings deferred from in-cycle PRs. Each item includes the
PR that surfaced it, severity at review time, and a short note on what

## From PR #43 (Task C — lifecycle bridge)

Resolved in commit on `feat/v1.2-lifecycle-bridge` before merge:

- 🔴 dashboard `fetchLifecycleJobs` read `window.SM_SELECTED_SESSION_ID`
  but Task B sets `PHASE6.selectedSessionId`. Pane never scoped to the
  picker. **Fixed**: now reads `PHASE6.selectedSessionId`.
- 🟡 `LifecycleBridge._seen` grew unbounded. **Fixed**: evict the
  matching start/end key pair when an `*_end` / `*_done` event lands.
- 🟡 dedup race: lock dropped before `bus.publish`. **Fixed**: lock now
  held across the publish call.
- 🟡 `/api/lifecycle/jobs` echoed raw exception strings to clients.
  **Fixed**: response body returns `"internal error"`; detail goes to
  the server log only.
- 🔵 `Message` lazy import in `_publish` hot path. **Fixed**: moved to
  module top.
- 🔵 dead `server._bus = bus` in test fixture. **Fixed**: dropped.

Open / deferred:

- 🟡 `list_active_jobs` overfetches with `LIMIT (limit * 4)` as
  "dedup headroom". On a hot session with >25 lifecycle event pairs
  (default limit=100, 2 rows per pair), open jobs at the tail get
  silently dropped. Replace with a windowed query (latest event per
  `job_id` via correlated subquery or window function) or paginate.
- 🔵 `HookFolderPoller` rotation detection uses `(st_ino, st_mtime)`.
  `st_ino` is unstable on Windows; rotation handling is best-effort
  there. Either add a content-hash sentinel of the first 64 B or note
  POSIX-only support in the docstring.
- 🔵 `_record_event_type` heuristic treats any record with
  `type == "tool_result"` and `background=True` as a completion. Low
  practical impact (tool_result rarely carries `background`), but a
  same-tick start/end pair could re-order. Tighten classifier.
- 🔵 tests reach into `bus._conn.execute(...)` directly. Add a public
  iterator on `MessageBus` or a read-only fetch helper and migrate the
  tests off the private attribute. Callers as of v1.2:
  `tests/test_lifecycle_bridge.py`, `tests/test_desktop_command_sse.py`
  (added in PR #44), `tests/test_desktop_commands.py`. Single sweep
  when the helper lands.

## From PR #44 (Task D — long-poll removal)

Open / deferred:

- 🔵 `CommandConsumer._process_one` is the test seam after `run_once`
  was deleted with the long-poll path. Underscore prefix suggests
  internal-only; keep it that way OR rename to a public
  `process_row(...)` if call sites grow. Until then, just a comment in
  the test file.
- 🔵 `_VALID_TRANSPORTS = frozenset({"sse"})` plus the explicit
  `transport == "long-poll"` raise is two checks for one removal.
  Future removals could share a `_REMOVED_TRANSPORTS = {name: msg}`
  lookup. Not load-bearing for v1.2.
