## Seed items

### 🟡 Haiku fastpath confidence floor — lever fires 0% under production load

**Added:** v1.8 P2 ship-gate close-out (2026-05-06)

**Why:** v1.8 P1 wired `is_ambiguous_block` / `is_hitl_synthesis` at the pre-routing call site. Unit tests confirm routing is correct. Ship-gate soak (60 events, 32.1 min, `reports/soak-20260506T101746Z.md`) recorded `cli_dispatch_fallback_ms` p95 = 0.00 and 0 `governance_fallback_routed` envelopes. Root cause: Haiku returns confidence ≥ 0.70 on all tested ambiguous-block content (short imperative destructive commands, deliberative question forms). The `BRIDGE_L4_FALLBACK_CONFIDENCE` floor (0.70 default) is never crossed. The Haiku-first path IS being taken (routing is wired), but the Sonnet retry never fires.

Two P2a attempts (deliberative questions → P1c-B; imperative declarative forms → P2a) both produced 0 fire rate. Third attempt not warranted without a mechanism change.

**Options for v1.9:**
1. Lower `BRIDGE_L4_FALLBACK_CONFIDENCE` from 0.70 to ~0.50 — risks always-fallback if Haiku returns 0.50-0.70 uniformly on borderline content (must measure first).
2. Restrict `is_ambiguous_block` to HITL-synthesis-only path — the HITL trigger is more deterministic (based on session state, not content heuristic); more reliable lever activation.
3. Replace confidence-based fallback trigger with verdict-based: if Haiku returns BLOCK on ambiguous-block content, auto-retry with Sonnet for a second opinion (more predictable trigger than confidence floor).
4. Expand the `is_ambiguous_block` heuristic to broader content patterns that Haiku genuinely finds borderline (requires empirical study of Haiku confidence distribution on governance prompts).

**Scope:** `src/stream_manager/cli_governance.py` (floor constant), `src/stream_manager/governance.py` (heuristic), `tools/soak_driver.py` (corpus). Alignment-eval `--ci-gate` re-run required.

**References:** `docs/adr/ADR-5-latency-budget.md` §"v1.8 ship-gate baseline" §"Caveats", `reports/soak-20260506T101746Z.md`, v1.8 P2a `docs/prompts/v1.8-orchestration/phase-2a-soak-prompt-coverage.md`.
