### 🟢 PPP audit harness — Provenance Probe Protocol

**Why:** SM today binds gov decisions to a JSONL tail + bus stream by path
+ project_slug; no runtime check that the stream SM watches is the one the
user is actually driving. Failure modes today are silent: wrong-session
bind, mirror-frame collision (sync-comms v1.0), self-monitor leak (per
`feedback_no_self_monitor.md`), parser hallucination on stale buffers.
Manual audit during v1.6 ship-gate soak surfaced this as a generic gap, not
a v1.6-specific bug — defer to v1.7.

**How to apply:** new envelope pair `audit.probe` / `audit.probe_ack`. Three
layers:
1. **Stream disambiguation** — HITL question lists top-K streams SM is
   watching (slug, jsonl_path, brain_id, last_event_ts, prompt_hash); user
   picks correct stream or "none". Cached as HMAC-signed provenance
   assertion, TTL ~30 min. On-demand `/sm-probe` + auto every 30 min during
   soak.
2. **Canary echo** — SM emits nonce; user types nonce in active session;
   SM observes nonce in claimed JSONL within 10s ⇒ binding proven both
   directions.
3. **Negative-control fake stream** — synthetic JSONL never written; if
   dashboard ever reports activity on it, parser is hallucinating events.
   Self-monitor guard: candidate-list filter excludes any JSONL whose
   `brain_id` matches SM's own brain.

Cassette must cover new envelope pair same cycle (per
`feedback_cassette_must_cover_new_envelopes.md`). Sonnet categorizer NOT
involved (advisory only). Depends on sync-comms v1.0 HITL panel landing
first; mark blocked-by until then.

Scope hint: multi-phase. ~3–4 sub-PRs (envelope wire-up; HITL panel
section + HMAC cache; canary loop; decoy stream + self-monitor guard).
Originating phase: v1.6 ship-gate finalize (S2 audit conversation).
