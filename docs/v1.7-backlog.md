### 🟡 Haiku fastpath — primary v1.6-driver lever

**Why:** v1.6 ship-gate (`reports/soak-20260505T073943Z.md`) localised the
`_evaluate_inner` p95 tail to `cli_pool_send_ms` p95 = 6328.07 ms (99.99%
of `cli_dispatch_ms` 6329.00 ms; ~99.98% of `evaluate_inner` 6329.38 ms
— see ADR-5 §"v1.6 ship-gate baseline"). Driver is the synchronous
`worker.send` Anthropic CLI round-trip at `cli_pool.py:255`, i.e.
upstream model wall-clock — local plumbing (`cli_setup_ms` 0.01 ms,
`cli_pool_acquire_ms` 0.06 ms, `cli_parse_ms` 0.15 ms) is negligible.
L2/L3 already routes through Haiku per `model_router.py`; Sonnet runs on
L4 / ambiguous-BLOCK. Downgrading more of the L4 / ambiguous-BLOCK path
to Haiku attacks the upstream model TTFT/decode component directly.

**How to apply:** revisit `model_router.py` band → model mapping; widen
the Haiku surface (e.g. fast-path L4 cases that don't need Sonnet
chain-of-thought; introduce confidence-gated Haiku-first / Sonnet-fallback
on ambiguous-BLOCK). Soak success criterion: ALLOW p95 + L4 p95 trend
down vs v1.6 baseline; LM categorize unchanged (separate code path);
`cli_pool_send_ms` p95 drops because more events take the cheaper model.
Cassette must cover any new routing branches same cycle (per
`feedback_cassette_must_cover_new_envelopes.md`).

**Risk — ship-blocker if not addressed:** alignment quality on L4. A
Haiku downgrade can silently regress safety on the band that mattered
most for v1.5 protected-symbol policy. **Required gate:** an alignment
eval harness (golden-set L4 prompts with known correct verdicts) must
land in the same cycle and pass at ship-gate before the router change is
allowed to merge. If alignment eval shows regression, the lever is
abandoned and pool sizing >2 (item below) becomes primary.

Scope hint: single-cycle, likely 1–2 PRs (router change + alignment eval
harness). Originating phase: v1.6 ship-gate finalize (S4 driver
localization). Carries v1.6 ADR-5 §"v1.6 ship-gate baseline" §Caveats
bullet 1.

### 🟢 CLI pool sizing >2 — concurrent-burst fallback lever

**Why:** v1.6 ship-gate showed `cli_pool_acquire_ms` p95 = 0.06 ms under
sequential soak workload — zero queueing — so raising
`--cli-pool-size` from the current 2 will not move p95 on the existing
soak driver. Pool sizing only pays off under genuine concurrent burst
load (e.g. multi-session sync-comms v1.0 traffic, multi-tab dashboard
fan-out). Carry as **insurance**, not a measurable v1.7 p95 lever.

**How to apply:** parameterise pool size beyond the current ship-gate
default; add a v1.7 burst-load soak tier (extending ADR-17) that drives
N>1 concurrent sessions and measures `cli_pool_acquire_ms` under that
shape. If sequential-soak `cli_pool_acquire_ms` stays ≤ 1 ms, this item
stays low-priority. Promote to 🟡 only if sync-comms v1.0 lands and
shows acquire-time elevation.

Scope hint: 1 PR (config plumbing) + ADR-17 amendment for new tier. Can
be deferred indefinitely. Originating phase: v1.6 ship-gate finalize (S4
fallback lever).

### 🟢 PPP audit harness — Provenance Probe Protocol

**Why:** SM today binds gov decisions to a JSONL tail + bus stream by path
+ project_slug; no runtime check that the stream SM watches is the one the
user is actually driving. Failure modes today are silent: wrong-session
bind, mirror-frame collision (sync-comms v1.0), self-monitor leak (per
`feedback_no_self_monitor.md`), parser hallucination on stale buffers.
Manual audit during v1.6 ship-gate soak surfaced this as a generic gap, not
a v1.6-specific bug — defer to v1.7.

**How to apply:** new envelope pair `audit.probe` / `audit.probe_ack`. Three
layers:
1. **Stream disambiguation** — HITL question lists top-K streams SM is
   watching (slug, jsonl_path, brain_id, last_event_ts, prompt_hash); user
   picks correct stream or "none". Cached as HMAC-signed provenance
   assertion, TTL ~30 min. On-demand `/sm-probe` + auto every 30 min during
   soak.
2. **Canary echo** — SM emits nonce; user types nonce in active session;
   SM observes nonce in claimed JSONL within 10s ⇒ binding proven both
   directions.
3. **Negative-control fake stream** — synthetic JSONL never written; if
   dashboard ever reports activity on it, parser is hallucinating events.
   Self-monitor guard: candidate-list filter excludes any JSONL whose
   `brain_id` matches SM's own brain.

Cassette must cover new envelope pair same cycle (per
`feedback_cassette_must_cover_new_envelopes.md`). Sonnet categorizer NOT
involved (advisory only). Depends on sync-comms v1.0 HITL panel landing
first; mark blocked-by until then.

Scope hint: multi-phase. ~3–4 sub-PRs (envelope wire-up; HITL panel
section + HMAC cache; canary loop; decoy stream + self-monitor guard).
Originating phase: v1.6 ship-gate finalize (S2 audit conversation).
