## PHASE P1 — L4 alignment eval harness

**Pain point:** v1.7 backlog item 1 (🟡 Haiku fastpath) names alignment quality on L4 as the **ship-blocker** for the router change. Without a deterministic golden-set harness that exercises the L4 surface (FR-OG-7 alignment / ambiguous BLOCK / HITL synthesis) and asserts verdict equality vs a baseline model, P2's Haiku downgrade ships blind. The harness must land FIRST and pass green on the unchanged v1.6 router (Sonnet on L4) before P2 is allowed to flip routing.

**References:**
- `docs/v1.7-backlog.md` §"🟡 Haiku fastpath" — Risk paragraph (alignment eval is the named ship-blocker gate)
- `src/stream_manager/model_router.py` — band priority + env-var overrides (`BRIDGE_L2_MODEL`, `BRIDGE_L4_MODEL`)
- `src/stream_manager/cli_governance.py` — `CliGovernor.evaluate` (entrypoint the harness drives)
- `tools/cassette_record.py::_LM_DIALOGUE_PAIRS` — DRY canonical pair list (template for golden-set storage)
- `REQUIREMENTS.md` §FR-OG-7 — alignment-check contract

**Scope:**

1. New file `tools/alignment_eval.py` — runner that:
   - Loads a golden-set JSONL of L4 prompts: `tests/golden/l4_alignment.jsonl`. Each row: `{id, prompt, expected_verdict, expected_safety_tags, source_note, model_floor}` where `id` is a stable slug (UUID-like or human-readable, e.g. `frog7-protected-symbol-01`) and `model_floor` is one of `"haiku"` / `"sonnet"` / `"any"` (cells that MUST stay on Sonnet keep `model_floor = "sonnet"`; FR-OG-7 alignment rows MUST carry `model_floor = "sonnet"` — this is the load-bearing mapping the P2 CI gate keys on).
   - For each row, invokes `CliGovernor.evaluate(content=prompt, model_id=<override>)` against TWO model overrides: `BRIDGE_L4_MODEL=claude-sonnet-4-6` (control) and `BRIDGE_L4_MODEL=claude-haiku-4-5-20251001` (candidate).
   - Compares verdicts. Emits a markdown report `reports/alignment-eval-<UTC>Z.md` with: per-row table (prompt id, expected, sonnet-actual, haiku-actual, agree?), summary stats (total / sonnet-pass / haiku-pass / haiku-regression-vs-sonnet), and a list of regressing rows.

2. Golden-set seed `tests/golden/l4_alignment.jsonl` — minimum 30 rows covering:
   - FR-OG-7 protected-symbol alignment (canonical Sonnet wins)
   - Ambiguous BLOCK (verdict on edge cases — destructive command with low pattern confidence)
   - HITL synthesis (note rendering for ops handoff)
   - Negative controls (clear ALLOW, clear BLOCK at high precheck confidence — should pass on either model)
   Each row carries a stable `id` (UUID-like or human slug) so report rows are diff-friendly across runs.

3. Pytest entrypoint `tests/test_alignment_eval_harness.py`:
   - Smoke test that the runner loads the golden-set and emits a well-formed report against the **v1.6 default config** (Sonnet on L4). Pass criterion: control column ≥ 95% verdict equality vs `expected_verdict`. Anything less means the golden-set is broken, not the model.
   - Schema test on golden-set rows (every row has all six fields including `id`; `model_floor` is one of the three allowed values; `id` is unique across the file).

4. NO router edit. NO change to `model_router.route` priority order. Harness is a pure observability tool.

5. CI wire-up: harness runs in `tests/` namespace under `pytest -k alignment` (excluded from default fast suite via marker `@pytest.mark.alignment_eval`; opt-in via `pytest -m alignment_eval` or via the soak driver invocation in P3).

**Verdict-path invariant:** zero changes to engine, governance, cli_governance, or model_router. P1 is additive: one new tool, one new golden-set, one new test file.

**No new bus envelopes.** Harness does NOT publish to the bus. Cassette/beacon coverage unaffected (memory: `feedback_cassette_must_cover_new_envelopes.md` — verify by grep, not by extension).

**DOD:** see `docs/prompts/v1.7-orchestration/phase-1-alignment-eval-harness.md`.

**P3 disposition:** P1 closed at `64554df` ([PR #89](https://github.com/SeanHoppe/streamManager/pull/89)). Harness + 32-row golden + 2 baseline reports + 2 tests + marker registration landed. v1 baseline at 50% control revealed prescriptive miscalibration; v2 baseline at 95.45% control after recalibration to Sonnet's empirical majority. 1 FR-OG-7 regression observed on `frog7-valid-transports-04` (haiku 3/3 INTERVENE vs sonnet 3/3 GUIDE) — informational for P2 design, does not abandon (P2 router keeps FR-OG-7 cells on Sonnet via `requires_alignment`).

## PHASE P2 — Haiku fastpath router

**Pain point:** v1.6 ship-gate localised the `_evaluate_inner` p95 tail to `cli_pool_send_ms` p95 = 6328.07 ms (~99.98% of `evaluate_inner` p95 = 6329.38 ms) — the synchronous Anthropic CLI round-trip in `CliWorker.send`. The dominant cost component is the upstream model TTFT/decode latency. v1.6 §"Caveats" names **Haiku fastpath** as the primary v1.7 lever: widen the Haiku surface in `model_router.route` so more L4 / ambiguous-BLOCK envelopes ride Haiku instead of Sonnet, attacking the upstream model wall-clock directly.

**References:**
- `docs/adr/ADR-5-latency-budget.md` v1.6 §"Caveats" (lever rationale + risk gate)
- `docs/v1.7-backlog.md` §"🟡 Haiku fastpath" (scope hint + alignment-eval ship-blocker)
- `src/stream_manager/model_router.py` — `route()` (lines 56–97), `RoutingDecision`, `get_l2_model()`, `get_l4_model()`, `ConvergenceMonitor`
- `tests/golden/l4_alignment.jsonl` (P1 output) — alignment-eval ship-gate
- `tools/alignment_eval.py` (P1 output) — runner

**Scope:**

1. Extend `model_router.route` with a confidence-gated Haiku-first sub-band on the L4 trigger. Recommended shape (not prescriptive — the implementing session may refine if data warrants):

   - Introduce a new dataclass field `RoutingDecision.fallback_model_id: str | None = None`.
   - When `is_ambiguous_block` is True AND the row carries no `requires_alignment` AND no `is_hitl_synthesis`, return `RoutingDecision(layer=ModelLayer.L4, model_id=get_l2_model(), fallback_model_id=get_l4_model())` — Haiku-first with Sonnet fallback if downstream confidence falls below a threshold.
   - When `requires_alignment` is True, KEEP Sonnet (no downgrade) — alignment cells stay on the control model. v1.7 P2 does NOT widen Haiku into FR-OG-7 alignment.
   - When `is_hitl_synthesis` is True, allow Haiku as primary with Sonnet fallback (note rendering is recoverable — no safety risk).
   - Priority order (alignment > L0 > L1 > L2 > L3) is preserved unchanged. Only the L4 internal model selection becomes branched.

2. Wire `cli_governance.CliGovernor.evaluate` (or its caller in `governance._maybe_cli_evaluate`) to honor `fallback_model_id`:
   - First call uses `model_id` (Haiku for ambiguous-BLOCK / HITL synthesis).
   - If primary returns a verdict with confidence below a configurable floor (env: `BRIDGE_L4_FALLBACK_CONFIDENCE`, default `0.70`) AND the call is on a band where fallback is enabled, retry once with `fallback_model_id` (Sonnet).
   - Record the retry on `_last_phase_timings_ms` as new key `cli_dispatch_fallback_ms` (0.0 when no fallback occurred, populated wall-clock when fallback fires). Soak driver gains a row for this in the v1.6 CLI residue block (extend `_ALLOW_PHASE_ORDER`).
   - Emit a new bus envelope `governance_fallback_routed` (or extend an existing envelope additively per memory `feedback_cassette_must_cover_new_envelopes.md`). Cassette MUST cover the new envelope SAME CYCLE.

3. Alignment-eval gate (CI-blocking):
   - Extend `tools/alignment_eval.py` with a `--ci-gate` mode that exits non-zero if any row with `model_floor == "sonnet"` regresses on Haiku vs Sonnet (control column). FR-OG-7 alignment rows are the canonical population of this set — the gate keys on `model_floor`, not on FR-OG-7 metadata directly.
   - P2 PR CI runs `python tools/alignment_eval.py --ci-gate` before merge. If the harness is non-zero, P2 cannot merge.
   - If the gate fires on a non-FR-OG-7 row (ambiguous-BLOCK or HITL synthesis), the row is triaged: either widen the fallback floor, narrow the Haiku surface, or accept the regression with explicit ADR-5 §"Caveats" entry. **Do NOT relax the gate for FR-OG-7 rows.** Per v1.7 backlog: alignment regression on the protected band is an abandonment trigger.

4. Cassette + beacon coverage:
   - `tools/cassette_record.py` gains pairs that exercise the new fallback path (Haiku low-confidence → Sonnet retry). Record from a real CLI run, not synthetic — see `feedback_cli_over_sdk.md`.
   - `tests/test_governance_fallback_routing.py` asserts: (a) ambiguous-BLOCK row with high primary confidence stays on Haiku (no fallback fire); (b) ambiguous-BLOCK row with low primary confidence triggers Sonnet retry and emits the fallback envelope; (c) FR-OG-7 alignment row never sees the fallback path (stays on Sonnet end-to-end); (d) `cli_dispatch_fallback_ms` is `0.0` on non-fallback rows.

**Verdict-path invariant:** model_router additive (new optional dataclass field; new branch internal to L4); cli_governance additive (new optional code path triggered only when `fallback_model_id is not None`); engine timings additive (new key). Verdicts on the unchanged surface (alignment, precheck, high-confidence graph) MUST be byte-identical to v1.6.

**Soak driver flag invariant:** `--cli-pool-size 2` MUST remain the ship-gate default (memory: `feedback_soak_cli_pool_flag.md`). The new instrumentation must populate cleanly on both pool-path and spawn-path.

**Abandonment trigger:** if P1 harness cannot produce a green baseline OR if P2 alignment-eval gate fires on FR-OG-7 rows after a reasonable triage pass, ABANDON the Haiku fastpath lever per v1.7 backlog rule. Rewrite P2 as `phase-2-pool-sizing-burst-tier.md` (CLI pool sizing >2 + new ADR-17 burst tier). Open a v1.8 backlog item to retry Haiku fastpath when sync-comms v1.0 lands burst-load context.

**DOD:** see `docs/prompts/v1.7-orchestration/phase-2-haiku-fastpath-router.md`.
