# v1.3 soak — Learn Mode envelope extension

**Branch:** `ship/v1.3-soak-lm-extension`

**Pain.** v1.3 P5 shipped Learn Mode (FR-LM-1..6: JSONL-tail dialogue ingest, Sonnet categorizer, advisory bias, decay) but the soak surface (`tools/cassette_record.py`, `tools/soak_driver.py`) was not extended. P6 M2 wording (`docs/v1.3-task-plan.md` §P6): *"refreshed cassette must include Learn Mode envelopes"*. Today both tools call `engine.evaluate` only — neither drives `desktop_prompt`/`user_reply` through the tailer or invokes the categorizer. M2 against today's recorder produces a cassette identical in shape to v1.2's; M3 ship-gate exercises zero of the new v1.3 code paths.

**Goal.** Extend recorder + driver so cassette + ship-gate exercise the Learn Mode hot path. Cassette format gains a `learn_dialogue` kind. Soak report gains an "LM (categorize)" per-band row. Backward-compatible with v1.2 cassettes.

## Design

### Cassette envelope kind: `learn_dialogue`

Adds one new value to the existing `kind` enum. Schema:

```json
{
  "idx": 60,
  "kind": "learn_dialogue",
  "content": "<desktop_prompt verbatim>",
  "recorded_latency_ms": 1234.5,
  "decision": {
    "action": "ALLOW",
    "confidence": 0.85,
    "reasoning": "category=approve",
    "matched_hash": "",
    "model_used": "claude-sonnet-4-5",
    "layer": 0
  },
  "desktop_prompt": "<verbatim>",
  "user_reply": "<verbatim>",
  "recorded_categorize_latency_ms": 1234.5,
  "category_result": {
    "category": "approve|reject|redirect|clarify|acknowledge|unknown",
    "confidence": 0.0-1.0,
    "reasoning": "<short>"
  }
}
```

Required-field audit (`_load_cassette` in `tools/soak_driver.py`): `kind`, `content`, `recorded_latency_ms`, `decision` already present → existing validation passes. Old replay code that doesn't recognise `learn_dialogue` would treat it as a routine envelope and write an ALLOW decision. New replay code routes by `kind`.

Backward compat: v1.2 cassettes have no `learn_dialogue` rows. Replay walks the cassette, the `kind` branch is never hit, LM band shows n=0 in the per-band table.

### Pre-canned dialogue pairs

10 pairs, deterministic, covering all 6 categories. Lives in `tools/cassette_record.py` as `_LM_DIALOGUE_PAIRS` and is shared with `soak_driver.py` via the same import idiom that already wires `_build_payload_sequence`.

| idx | desktop_prompt (truncated) | user_reply | expected category |
|---|---|---|---|
| 0 | "Force push to main now?" | "yes do it" | approve |
| 1 | "Run rm -rf node_modules?" | "no, keep them" | reject |
| 2 | "Install pkg X to fix Y?" | "use pkg Z instead" | redirect |
| 3 | "Run pytest in CI?" | "what does the suite cover?" | clarify |
| 4 | "Branch is rebased and pushed." | "ok thanks" | acknowledge |
| 5 | "Deploy to staging?" | "approved, ship it" | approve |
| 6 | "Drop user_settings table?" | "absolutely not" | reject |
| 7 | "Use pickle for serialization?" | "use json instead" | redirect |
| 8 | "Run the migration?" | "explain what it does first" | clarify |
| 9 | "All checks green." | "👍" | acknowledge |

The `unknown` slot is implicit — the categorizer may emit it on any mismatched parse, so the table doesn't pre-allocate one.

### Recorder (`tools/cassette_record.py`)

After the existing 60-envelope `engine.evaluate` loop:

1. For each pair in `_LM_DIALOGUE_PAIRS`:
   - `bus.publish(desktop_prompt)` + `bus.publish(user_reply)` with proper `parent_uuid`/`pair_id` linkage (mirrors `jsonl_tail._maybe_emit_learn_mode`).
   - Call `learn_categorizer.categorize_pair(prompt, reply, model=DEFAULT_MODEL)` directly — bypass the worker poller for deterministic latency capture.
   - Write a `learn_dialogue` cassette row with the recorded latency + result.
2. Failures (None result) produce a row with `category="unknown"`, `confidence=0.0`. Same poison-skip behaviour as the live worker.

### Driver replay path (`tools/soak_driver.py::_run_replay`)

- Add `kind == "learn_dialogue"` branch:
  - Sleep `recorded_categorize_latency_ms`.
  - Call `bus.record_decision` with the synthesized decision exactly as today (so SSE consumer counts match).
  - Append latency to `state.lm_categorize_latencies_s`.
- New per-band row "LM (categorize)" in `_format_per_band_split`.

### Driver ship-gate path (`tools/soak_driver.py::main`)

- After the 60-envelope loop, run the same dialogue pump as the recorder. Real Sonnet calls.
- Track LM band latencies. Same per-band row.
- Honor `--skip-lm-pump` flag for legacy CI runs without quota.
