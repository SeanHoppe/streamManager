## NEW v2.4 ship-gate seeds (filed at P2)

5. ✅ **Seed v2.4-O — soak `_git_diff_loc` whole-repo bug. RESOLVED PR #184 (squash-merge `9ca5226`, 2026-05-19).**
   `tools/soak_driver.py` `_git_diff_loc` previously ran
   `git diff --shortstat ANCHOR..HEAD` without a path filter,
   violating Amendment C's production-bucket scope (`src/ + tests/
   + tools/ + dashboard/`). v2.4 P2 soak summary recorded phantom
   **BLOCK** (+747 / -1 / +746 vs cycle-tip) because docs + reports
   changed since cycle-tip. Bucket-scoped manual diff = **0 net** =
   PASS. Bug latent in v2.3 (feature-cycle 2250 tolerance hid it);
   first consolidation cycle to surface the helper bug.

   **Fix RESOLVED in PR #184 (v2.4 P2 ship-gate review-fix):**
   `_git_diff_loc` now accepts `paths: list[str] | None` +
   `require_paths: bool` kwarg; both ship-gate call-sites
   (`_write_report`, `main`) pass `require_paths=True` so a missing
   `BRIDGE_LOC_PATHSPEC` yields a loud `PATHSPEC-UNSET` sentinel
   (→ gate verdict UNKNOWN) instead of silently reverting to
   whole-repo. Tests: 42 cases at
   `tests/test_soak_summary_loc_anchors.py` covering pathspec
   forwarding, env parser, scope × tolerance matrix per
   [[feedback_cycle_tolerance_masks_bugs]], `require_paths`
   sentinel, e2e env-to-argv threading through both call-sites.
   No longer carries to v2.5; this row stays as historical record
   only.

6. 🟡 **Seed v2.4-P — P2 prompt S1 wipe step deletes tracked
   reports.** The S1 step's `Remove-Item -Force reports/soak-*.md`
   wildcard literally deletes tracked historical soak reports from
   working dir on Windows PowerShell; first run of v2.4 P2 surfaced
   ~36 phantom deletes that contaminated the soak-driver's already-
   buggy `_git_diff_loc` output. `git checkout HEAD -- reports/`
   recovered on the second run.

   **Fix RESOLVED in PR #184 (v2.4 P2 ship-gate review-fix):** S1
   now uses `git clean -df reports/` instead of a filename glob.
   `git clean -df` removes ONLY untracked files under `reports/` by
   construction, so tracked baseline reports are preserved
   regardless of the driver's output naming. The earlier PR #184
   first-draft narrowed the glob to `reports/soak-tmp-*.md`, but
   review found that pattern matches NOTHING against the real
   driver output (`tools/soak_driver.py:1648` writes
   `reports/soak-{iso_ts}.md`, no `tmp-` prefix) — a no-op
   masquerading as a fix. The final `git clean -df` mechanism
   sidesteps the filename-pattern question entirely. S1.1 post-wipe
   assertion now uses `git status --short --untracked-files=no` so
   stray untracked files (cassettes, scratch) cannot false-fire
   BLOCK before S2. Pairs naturally with Seed v2.4-O closure (once
   the helper is bucket-scoped, false LOC deltas from a wipe stop
   mattering regardless).
