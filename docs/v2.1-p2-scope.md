## 1. Goal

Prove the JSONL stream binding from **both directions** using a
nonce echo:

1. After operator signs a Layer-1 assertion (`provenance_assertions`
   row), SM emits an `audit.canary_emit` envelope carrying a freshly
   generated nonce + probe_id + target `jsonl_path` (from the
   assertion's `selected_jsonl_path`).
2. Dashboard surfaces the nonce to the operator with a "type this
   string into your active session" prompt.
3. Operator types the nonce in the active Claude CLI session.
4. SM's `JsonlTailWorker` observes the nonce in the claimed JSONL
   within a 10s window ‚áí binding proven in BOTH directions.
5. On observe: SM emits `audit.canary_observed` and marks the
   assertion `canary_confirmed_at = NOW()`.
6. On 10s timeout: SM emits `audit.probe_failure` (reason=`canary_timeout`)
   and re-fires Layer 1 disambiguation HITL (binding is suspect).

## 5. Threat model continuity

Layer 2 sig coverage:
- `audit.canary_emit` sig binds `{probe_id, jsonl_path, nonce, issued_at, timeout_s}` ‚Äî endpoint-integrity per FR-PPP-2.
- `audit.canary_observed` sig binds `{probe_id, nonce, observed_at, jsonl_path}` ‚Äî server-stamped at observe time inside `jsonl_tail.py` worker. The worker holds the `desktop_command` secret of record (already loaded for SSE-receipt validation).
- `audit.probe_failure` sig binds `{probe_id, reason, failed_at}` ‚Äî server-stamped on the failure path.

**Replay protection:** the canary nonce is single-use; once
`mark_canary_confirmed` records `(probe_id, nonce, confirmed_at)` the
registry entry is cleared. A second observe of the same nonce in the
JSONL (operator re-typed it) is logged but no second envelope is
emitted.

**Self-monitor guard:** the observer MUST NOT scan SM's own JSONL.
Reuse the `_is_sm_originated` filter from `JsonlTailWorker` ingest
path (already in place per FR-PPP-1 P1 ¬ßself-monitor preview). The
P1a `tests/test_audit_probe_self_monitor.py` candidate-list filter
remains in force at HITL emit time; the observer-side filter is
defense-in-depth.

## 8. Risk register (P2-specific)

| # | Risk | Mitigation |
|---|---|---|
| R1 | LOC overrun > 700 | Tighten dashboard surface via shared HITL row helper; if hard, split P2a per ADR-18 Rule 4 |
| R2 | Nonce observer races with tail seek-to-end on a JSONL roll | `JsonlTailWorker` already handles new-file detection at L100+; the canary registry's `target_jsonl_path` is the path-at-emit-time, so a roll mid-canary will fail to observe. Mitigation: accept the failure ‚Äî re-queue Layer 1 is the correct response when the binding has shifted |
| R3 | Self-monitor leak via observer scanning SM's own JSONL | Reuse `_is_sm_originated` ingest filter; M3 must call it before nonce match |
| R4 | Multiple concurrent canaries on the same session | Registry is keyed by probe_id (UUID); concurrency is safe. Operator UI shows one at a time but registry semantics permit overlap |
| R5 | Cassette gap recurs | M5 lands cassette + cassette test same-cycle (test 19) |
| R6 | 10s timeout too tight under operator hesitation | M3 should make timeout configurable per emit (default 10s, dashboard auto-emit hook can override to 30s); cassette uses short timeout for determinism |
| R7 | Failure re-queue infinite loop (probe_failure ‚Üí Layer 1 ‚Üí canary ‚Üí probe_failure) | Cap re-queue depth at 1 per session in `emit_audit_probe_failure`; second failure surfaces a hard HITL warning, not a re-queue |
| R8 | `provenance_assertions` ALTER migration on a populated WAL | Migration follows v1.1 / v1.3 idempotent `PRAGMA table_info` pattern at `message_bus.py:389+` (Task L precedent); existing rows get `canary_nonce=NULL, canary_confirmed_at=NULL` ‚Äî semantically equivalent to "P1 row, no canary attempted" |
