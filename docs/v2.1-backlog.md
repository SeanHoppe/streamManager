### 🟡 `--total-events` flag drift — ADR-17 vs `soak_driver.py`

P2) reference a Tier 1.5 invocation using `--total-events 6`. The
flag does NOT exist in `tools/soak_driver.py` — only `--total-seconds`
+ `--interval-seconds` are accepted. Discovered while running the
v2.0 P3 Tier 1.5 smoke soak. Workaround used:
`--cli-pool-size 2 --total-seconds 120 --interval-seconds 20`.

Disposition options:
- **Option A**: add `--total-events` flag to `tools/soak_driver.py`
  matching ADR-17 + matrix wording (operator wins; matrix is the
  intended UX).
- **Option B**: amend ADR-17 + matrix to reference the existing
  `--total-seconds` form (drop the flag from docs).

Out of scope for v2.0 (ADR-18 Rule 4 phase cap reached). One-PR
disposition in v2.1.
