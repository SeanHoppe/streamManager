**M3 design constraints (folded from M1 caveman-review):**

- **TOCTOU fix:** `/api/sm-probe?force=1` handler MUST branch on `bus.write_envelope(...)` return value (== 0 → HTTP 503), NOT pre-check `bus.envelope_subscriber_count()`. Subscribers can register/disconnect between the count read and the envelope dispatch. The `envelope_subscriber_count()` accessor is telemetry-only.
- **Add `unsubscribe_envelope(callback)`** to `MessageBus` in M3 same time as SSE per-connection subscribers wire up. Without it, browser disconnects leak callbacks.
- **SSE bridge:** dashboard must register one envelope subscriber per SSE client connection, scoped via `try/finally` to call `unsubscribe_envelope` on connection close (or generator cleanup).
