## Carry-forward from v1.3.1 ship-gate (PR #75 / Path-A)

- 🟡 **ALLOW p95 publish-path instrumentation**
  v1.3 ship-gate produced the first separated ALLOW p95 measurement
  (`reports/soak-20260504T152005Z.md` — 9.60 s, n=50). The v1.2 ADR-5
  speculation that ALLOW p95 was "likely well inside the ≤ 6 s budget"
  was wrong; v1.3 ADR-5 widened the budget to ≤ 12 s pending diagnosis.
  Working hypothesis (ADR-5 §"Caveats"): the ALLOW band traverses the
  full `engine.evaluate` → `decision_graph.classify` → `bus.publish`
  → `record_decision` chain, and the upper tail is dominated by
  sqlite3 contention in the publish path under the 30-min wall-clock
  window — NOT by an L0 → L2 misroute. v1.4 should instrument the
  publish path (per-call wall-clock breakout: classify / publish /
  record_decision / commit), then decide whether to tighten ALLOW p95
  back toward ≤ 6 s or widen permanently.
  Originating: v1.3.1 ADR-5 v1.3 ship-gate baseline §"Caveats".

- 🔵 **Cassette LM coverage in beacon harness**
  PR #75 Path-A cassette ships 10 fixed `learn_dialogue` pairs in
  `tools/cassette_record.py::_LM_DIALOGUE_PAIRS` covering all 6
  category enum values. Consider integrating these with
  `tests/beacons/learn_mode_categorizer.jsonl` (P5e Method 2) so the
  same canned inputs drive both replay-tier regression detection and
  beacon-tier categorizer accuracy assertion. Today the two artifacts
  duplicate intent.
  Originating: PR #75 Path-A close-gaps cycle.
