## Compare-back protocol (per-landing)

When a v2.3 PR lands:

1. Find the seed row this PR addresses (use compare-back row marker).
2. Mark `[x]` and append `LANDED PR #N (<merge-SHA>)`.
3. If the PR addressed a seed that was NOT in the fire-order, append
   `OFF-SCOPE — operator-bound; recorded for narrative only`.
4. If the PR closed a 🔴 row → drop the 🔴 prefix in the v2.3 backlog
   carry-forward at v2.3 P2 close-out.
5. At v2.3 P2 ship-gate close memory, re-run row-by-row pass: every
   row must be ✅ LANDED, ⏸ DEFERRED v2.4 with rationale, or ❌
   DROPPED with rationale.

Strict invariant: **no v2.3 PR is "done" until this file is updated**.
