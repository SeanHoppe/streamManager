## Cycle-type fallback (consolidation override)

If operator overrides default-lean to **consolidation** at P0 fire,
the following dispositions apply:

- Both work-phase rows (`phase-1-cli-timeout-instrumentation.md` +
  `phase-1-shadow-synthetic.md`) become **DEFERRED v2.7**.
- Seed v2.5-G + Seed v2.5-C both rename to Seed v2.6-G / v2.6-C and
  carry to v2.7 P0.
- Cycle becomes 2-numbered-phase (P0 + P2 ship-gate), 0 work phases.
- Lever ledger HOLD at 1 production / 0 soak (unchanged).
- Operator records the override rationale in `docs/v2.6-task-plan.md`
  §"Operator decisions" #1 (typical: scope uncertainty, dependency
  on v10.3 stochastic propensities not yet available, lever-budget
  bound, etc.).

This fallback exists because alternation hygiene is NOT law; ADR-18
Amendment A §"Default lean rationale" permits two-cycle-in-a-row of
the same type when the candidate-lever backlog has no ready entry.
Seed v2.5-G is ready; Seed v2.5-C is ready iff v10.3 dependencies
are accounted for. Operator judges at P0 fire time.

## Compare-back protocol (per-landing)

When a v2.6 PR lands:

1. Find the seed row this PR addresses (use compare-back row marker).
2. Mark `[x]` and append `LANDED PR #N (<merge-SHA>)`.
3. If the PR addressed a seed that was NOT in the fire-order, append
   `OFF-SCOPE — operator-bound; recorded for narrative only`.
4. If the PR closed a 🔴 row → drop the 🔴 prefix in the v2.6 backlog
   carry-forward at v2.6 P2 close-out.
5. At v2.6 P2 ship-gate close memory, re-run row-by-row pass: every
   row must be ✅ LANDED, ⏸ DEFERRED v2.7 with rationale, or ❌
   DROPPED with rationale.

Strict invariant: **no v2.6 PR is "done" until this file is updated**.
