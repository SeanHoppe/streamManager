### NEW

1. `src/stream_manager/latency_budgets.py` — single export
   `BRIDGE_FALLBACK_LATENCY_BUDGET_MS = 35_000` (ms). Comment
   anchors value to `cli_governance.TIMEOUT_SECONDS * 1.4` rounded
   up — provides headroom over CLI timeout + fallback path.
2. `tests/test_api_timeout_invariant.py` — two test functions:
   - `test_cli_timeout_degrades_to_observe_under_budget`
   - `test_cli_5xx_error_degrades_to_observe_under_budget`
   Each patches `src.stream_manager.cli_governance.CliWorker.send`
   to either hang past `TIMEOUT_SECONDS` (timeout case) or raise a
   simulated 500/502/503 stderr payload, then asserts (a) verdict
   == `OBSERVE`, (b) end-to-end latency < the LIVE
   `BRIDGE_FALLBACK_LATENCY_BUDGET_MS` value (imported at runtime,
   not literal-pinned).
