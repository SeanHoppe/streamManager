### 🟡 v1.9 P1b — cli_pool worker per-turn isolation A/B

**Carry-forward from v1.9 P1a outcome.** P1a probe showed 96%
BLOCK/INTERVENE on `_L2_L3_TRIGGER` content via fresh `claude -p`
subprocess, vs 0% in 1800s soak through `cli_pool`'s long-lived
stream-json workers. Hypothesis (untested): cli_pool worker reuse
across ~30 turns biases Haiku toward conversational ALLOW on later
destructive prompts despite per-turn `session_id` rotation.

Investigation scope:
- Instrument `CliWorker.send` with verdict-class A/B: worker-recycle
  every-N-turns (N ∈ {1, 5, 10}) vs reuse (status quo) on identical
  `_L2_L3_TRIGGER` corpus.
- Measure: `cli_dispatch_fallback_ms` p95 fire rate per recycle
  cadence, process-spawn p95 cost added per recycle.
- Decision: recycle-per-turn (cost: ~1.5s spawn × N), recycle-every-N
  (cost vs lever-fire trade), accept dormancy (caveat in ADR-5).

Refs: `reports/p1a-corpus-haiku-verdicts-20260507T075646Z.md`,
`reports/soak-20260506T230328Z.md`,
`docs/v1.9-task-plan.md` §"PHASE P1a outcome".

Do-not-touch: v1.1 Task J `cli_pool` API surface
(`CliPool`, `CliWorker`, `CliWorker.send` signature) — extend with
optional kwargs only. ADR-5 latency budget enforcement at ship-gate
must remain intact.

### 🟡 Tier 1.5 smoke soak — codification + trigger matrix

**Carry-forward from v1.9 plan-agent soak-cost review.** ADR-17
already defines 3 tiers but lacks: enforcement (no CI gate), trigger
rules (which PR runs which tier), and a "fast soak" variant between
Tier 1 (replay, 0 tokens, no `cli_pool`) and Tier 3 (32 min, full
budget).

Proposed Tier 1.5: `python tools/soak_driver.py --cli-pool-size 2
--total-events 6 --total-seconds 120` (~90s wall, ~6 Haiku calls).
Validates pool warmup + first-turn dispatch + clean shutdown. Misses
long-tail leaks and steady-state drift — binary "pool warmed and
shut clean" gate only, not a latency gate.

Investigation scope:
- Codify Tier 1.5 in ADR-17 alongside existing tiers.
- Author trigger matrix (PR-touch path → tier) and land in
  `docs/prompts/v2.0-orchestration/phase-X-ship-gate-finalize.md`
  (or wherever v2.0 ship-gate phase lives).
- Estimate token reduction at typical-cycle scale (~92% Haiku, ~94%
  Sonnet vs status-quo "every gate Tier 3").

Deferred from v1.9 because: v1.9 has unresolved signal-quality issues
(P1a hypothesis untested, P2/P3 brand-new code surfaces ~2800 lines
not yet exercised under sustained load). Cutting observation budget
before establishing v1.9 baseline risks masking real regressions.
Land in v2.0 P0 cycle frame when v1.9 baseline is solid + P1b data
informs whether cli_pool reuse changes verdicts.

Refs: plan-agent report 2026-05-07 (inline in conversation, not
committed), `docs/adr/ADR-17-soak-tiers.md`,
`tools/soak_driver.py`.

Do-not-touch: ADR-17 existing tier definitions, ADR-5 baseline
source-of-truth (Tier 3 only).
