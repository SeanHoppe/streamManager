## PHASE P1 — Content-detection wiring

**Pain point:** v1.7 P2 ([PR #90](https://github.com/SeanHoppe/streamManager/pull/90), merged `9c483be`) shipped the L4 sub-band Haiku-fastpath router infrastructure. v1.7 P3 ship-gate ([PR #91](https://github.com/SeanHoppe/streamManager/pull/91), merged `b7e61044`) recorded 0 fallback fires across 60 soak events because the production caller path at `src/stream_manager/governance.py::_evaluate_inner_core` (search for `pre_routing = route(...)`) sets both `is_ambiguous_block` and `is_hitl_synthesis` to `False` unconditionally. The L4 sub-band's Haiku-fastpath branch is therefore unreachable from production code; `cli_dispatch_fallback_ms` p95 = 0.00 ms; 0 `governance_fallback_routed` envelopes emitted. ADR-5 §"v1.7 ship-gate baseline" §Caveats records the lever falsification (cli_pool_send_ms p95 dropped 19% but is upstream Anthropic variance, not lever effect, because the lever code never executed).

**References:**
- `docs/v1.8-backlog.md` §"🟡 `is_ambiguous_block` / `is_hitl_synthesis` content-detection wiring" — full Why/How/Risk/Scope
- `src/stream_manager/governance.py` — pre-routing call site (`_evaluate_inner_core`); HITL classify trigger surface (`hitl_classify_trigger` timing key + companion methods)
- `src/stream_manager/model_router.py` — `route(requires_alignment, is_ambiguous_block, is_hitl_synthesis)` signature (v1.7 P2)
- `src/stream_manager/cli_governance.py` — `CliGovernor.evaluate(fallback_model_id)` retry path
- `tools/alignment_eval.py` + `tests/golden/l4_alignment.jsonl` — v1.7 P1 alignment-eval harness (the `--ci-gate` ship-blocker)
- `reports/soak-20260505T125741Z.md` — v1.7 baseline (the lever-effect comparison point: cli_pool_send_ms p95 = 5128.96 ms, ALLOW p95 = 5.13 s, L4 p95 = 13.41 s)

**Do-not-touch guard:** the v1.7 P2 router + cli_governance + governance + soak driver surface is now load-bearing. Verdict-path invariant: the wiring change MUST keep verdict equality on the unchanged surface (alignment-only rows, precheck-hit, high-confidence graph, L2/L3 graph misses) BYTE-IDENTICAL to v1.7. Diff PR head vs `main` on `src/stream_manager/governance.py` and confirm only the pre-routing call site + supporting helper(s) change. NO edits to `model_router.py`, `cli_governance.py`, or `tools/soak_driver.py` (the v1.7 surface is reused as-is).

**Scope:**

1. **`src/stream_manager/governance.py`** — wire content-detection at the pre-routing site:
   - Compute `is_ambiguous_block` from content. Heuristic candidates (pick one or layer two):
     - regex match on a curated destructive-pattern list (`rm -rf`, `git push --force`, `drop table`, `truncate`, `delete from`, `chmod 777`, etc.) at low precheck confidence (i.e. precheck did not hit but content has destructive intent)
     - learn-mode bias hit on a "likely-block" pattern
     - leverage existing `_classify_ops` machinery if it surfaces destructive-action signal pre-CLI
   - Compute `is_hitl_synthesis` from the existing HITL classify trigger surface — thread the signal that already gates `hitl_classify_trigger` timing into the pre-routing call.
   - Update the pre_routing.route(...) call to pass the computed flags. Existing `requires_alignment` flag stays untouched.
   - **Verdict-path invariant:** when both new flags are False (the v1.7 default state), behavior must be byte-identical to v1.7 — same `pre_routing.fallback_model_id is None`, same `_maybe_cli_evaluate(fallback_model_id=None)`, same evaluate path.

2. **Tests:**
   - `tests/test_governance_content_detection.py` (NEW) — unit tests:
     - destructive-pattern content sets `is_ambiguous_block=True` (and `pre_routing.fallback_model_id == get_l4_model()`)
     - non-destructive content keeps `is_ambiguous_block=False` (and `pre_routing.fallback_model_id is None` — v1.7 dormant default)
     - HITL synthesis context sets `is_hitl_synthesis=True` (and fallback wiring active)
     - alignment-required content keeps `requires_alignment=True` and `fallback_model_id is None` regardless of the other two flags (FR-OG-7 protected)
   - Existing v1.7 fast-tier tests (`test_governance_fallback_routing.py`, `test_model_router_l4_subband.py`, `test_cli_governance.py`, `test_model_router.py`, `test_soak_driver_v17_residue_block.py`, `test_alignment_eval_harness.py` schema) MUST stay green.

3. **Alignment-eval `--ci-gate`** — re-run against the new wiring; must exit 0 on FR-OG-7 rows (`requires_alignment` keeps them on Sonnet, so haiku never sees them; FR-OG-7 regression should be impossible by construction). Attach the report to the PR.

4. **Local soak smoke** — drive a short cassette/soak run with content that should trigger fallback (curated destructive prompts in the L2/L3/L4 mix). Verify:
   - `cli_dispatch_fallback_ms` p95 > 0 in the report (proves the lever fires)
   - At least one `governance_fallback_routed` envelope emitted (cassette schema unchanged — bus envelopes are out-of-cassette by design; verify via dashboard log or bus message capture)
   - Fallback fire rate < 30% on the load mix (per v1.8-backlog risk paragraph; > 30% means Haiku is mostly retrying and the latency win is minimal — mint `phase-1a-fallback-floor-tuning.md` to raise `BRIDGE_L4_FALLBACK_CONFIDENCE` if so)

**Verdict-path invariant:** zero changes to `model_router.py`, `cli_governance.py`, `tools/soak_driver.py`, or v1.7 protected symbols. P1 is additive: one wiring change in `governance.py` + one new test file.
