## 5. Threat model continuity

Layer 3 sig coverage:
- `audit.hallucination_detected` sig binds `{probe_id, jsonl_path,
  detected_at}` — server-stamped at detect time inside `jsonl_tail`
  worker (worker already holds the `desktop_command` secret for canary
  observed sigs). Endpoint-integrity per FR-PPP-2.
- `provenance_decoys` row HMAC sig binds `{probe_id, jsonl_path,
  registered_at}` — server-stamped at register time inside
  `governance.register_decoy_stream`. Confidentiality NOT required
  (anyone reading the WAL sees the synthetic path); integrity of the
  registration row is the requirement.

**Single-emit invariant:** the `provenance_decoys.triggered_at` column
is the single-write-wins lock. The first hallucination detection on a
(probe_id, jsonl_path) pair stamps `triggered_at`; subsequent matches
no-op via `WHERE triggered_at IS NULL` (pop-then-emit pattern from P2
R7 mitigation).

**Self-monitor guard preserved (defense in depth):** the P2 observer's
existing `_is_sm_originated` filter MUST still fire before any decoy
match. M4 places the decoy hook AFTER the filter in
`_process_line`.
