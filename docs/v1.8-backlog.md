### 🟡 `is_ambiguous_block` / `is_hitl_synthesis` content-detection wiring — activate v1.7 P2 dormant lever

**Why:** v1.7 P2 ([PR #90](https://github.com/SeanHoppe/streamManager/pull/90), merged
`9c483be`) shipped the L4 sub-band Haiku-fastpath router infrastructure
(`RoutingDecision.fallback_model_id` + cli_governance retry path + new
bus envelopes + 27 tests). However, the production caller path at
`src/stream_manager/governance.py` (search for the `pre_routing = route(...)`
call site at `_evaluate_inner_core`) sets `is_ambiguous_block=False` and
`is_hitl_synthesis=False` unconditionally, so the L4 sub-band Haiku-fastpath
branch is unreachable from production code. v1.7 ship-gate soak
(`reports/soak-20260505T125741Z.md`) recorded 0 fallback fires across
60 events; `cli_dispatch_fallback_ms` p95 = 0.00 ms; 0
`governance_fallback_routed` envelopes emitted. ADR-5 §"v1.7 ship-gate
baseline" §Caveats records the lever falsification (cli_pool_send_ms p95
dropped 19% but is upstream Anthropic variance, not lever effect).

**How to apply:** wire content-based detection of the two flags at the
pre-routing site:
  - `is_ambiguous_block` — likely heuristic on prompt content (BLOCK-keyword
    detection + low precheck confidence proxy) OR a two-pass design where
    a fast precheck classifies likely-BLOCK candidates before the expensive
    pre-routing call.
  - `is_hitl_synthesis` — already gated by `hitl_classify_trigger`
    elsewhere in the engine; thread that signal into the pre-routing
    call site.
After wiring, re-run the v1.7 ship-gate soak shape and measure:
  - `cli_dispatch_fallback_ms` p95 (should be > 0; non-zero = lever fires)
  - Fallback fire rate (target < 30%; per P3 §4 a higher rate would mean
    Haiku is mostly retrying so latency win is minimal — mint
    `phase-Xa-fallback-floor-tuning.md` to raise the floor if so)
  - `cli_pool_send_ms` p95 vs the v1.7 baseline (5128.96 ms) — earned
    improvement when fire rate is non-zero AND p95 drops further
  - L4 alignment p95 unchanged (FR-OG-7 cells still on Sonnet only)

**Risk — ship-blocker if not addressed:** alignment quality on cells
that newly route through Haiku. P1 alignment-eval harness must re-run
as a `--ci-gate` PR check after wiring; if FR-OG-7 rows regress (none
should, since `requires_alignment` keeps them on Sonnet), abandon and
revisit. `docs/v1.7-task-plan.md` abandonment rule applies recursively.

Scope hint: 1 PR (governance.py pre-routing wiring + heuristic
implementation + tests). Originating phase: v1.7 P3 ship-gate finalize
(lever-falsification-check finding). Carries v1.7 ADR-5 §"v1.7
