# Single Source of Truth -- determination + lightest mechanism

**Status:** Proposal / determination (operator ask, directive follow-up E).
**Question asked:** *"determine if a single SOT solution is needed to prevent drift and second-guessing."*
**Verdict:** A single hand-maintained SOT doc is NOT the answer -- it would become the (N+1)th drifting doc. What IS needed is (a) one canonical owner PER CONCERN, (b) a GENERATED status index that cannot drift independently, and (c) a drift-check that fails loudly. Argued below.
**Scope:** docs/process proposal -- touches no FROZEN code surface. ASCII-only (cp1252).

---

## 1. The drift is real and already documented

This is not hypothetical. The corpus already records its own drift:

- `docs/v10-mvp-status.md` is **STALE** -- "still says P5 0%/~80%" while real infra is ~95-96%
  (auto-memory `project_v10_p5_infra_landed.md`). A status doc that lies is worse than none.
- The ship-gate LOC anchor was **stale at `4902cca`** while the cycle tip had moved
  (`MVP-GAP-REPORT.md` / `project_mvp_workflow_suite_wired.md`).
- The WIRED_LEVER_LEDGER read **0 in one doc vs 3 in another** ("OQ-1 ledger 0-vs-3").
- The doc's "+1617" LOC conflated test scope vs the binding "+928"
  (`project_mvp_workflow_suite_wired.md`).

Each is a second-guessing event: the operator (or an agent) cannot trust a number without
re-deriving it. That is the cost the ask is trying to kill.

## 2. The corpus inventory (why "just write one doc" fails)

Authoritative-ish intent/state is spread across, at minimum:

| Concern | Today's home(s) |
|---|---|
| Product intent | `INTENT.md` |
| Requirements (FR-*) | `REQUIREMENTS.md` |
| Dev rules / firewall / polarity | `CLAUDE.md` |
| Coding nuances | `MEMORY.md` (project root) |
| Per-topic machine memory | ~45 files under `.../memory/` |
| Cycle history | `project_v*_cycle_close.md` memories + git tags |
| Surface freeze law | `docs/adr/ADR-18-*.md` + other ADRs |
| MVP gauge | `docs/v10-mvp-status.md` (STALE) |
| Gap findings | `MVP-GAP-REPORT.md` |
| Workflow design | `streamManagerworkflow.md`, `Claude-ResearchFixWorkflow.md` |
| Learn-mode design | `smartai.md`, `docs/learn-mode-design.md` |
| Latency baseline | `docs/adr/ADR-5-*.md` |

A new hand-maintained `SOT.md` that restates these would: (a) duplicate content already owned
elsewhere, (b) need manual sync on every cycle, (c) drift the moment one update is missed -- which
is exactly the failure mode above. It would ADD a doc to second-guess, not remove one.

## 3. Determination: NO single hand-maintained SOT. YES to three mechanisms.

### 3.1 One canonical owner per concern (cheap, mostly already true)

Make the ownership table in section 2 explicit + binding: each concern has exactly ONE authoritative
doc; every other doc that mentions it is **derived/historical** and says so in front-matter:

```
---
canonical: false
derived_from: INTENT.md, ADR-18
as_of: 2026-06-11
note: historical -- see canonical source for current truth
---
```

This converts "which doc is right?" (a search) into "read the front-matter" (a glance). No new SOT;
just a stamped ownership contract. The cycle-close auto-memories are inherently historical -- stamp
the index, not each file.

### 3.2 A GENERATED `STATUS.md` (the actual "single pane")

The thing the operator actually wants -- one place to see current state -- must be **generated from
machine-readable sources, never hand-edited**, so it physically cannot drift independently:

- **cycle / tag:** `git describe --tags` + latest `project_v*_cycle_close` -> current version + tip SHA.
- **MVP gauge:** derive from the real signal (ledger row count, `rl_episodes.db` episode count, soak
  pass artifacts) -- NOT a hand-typed percentage. Kills the `v10-mvp-status.md` stale-number class.
- **lever ledger:** read the single ledger source, print it once. Kills the 0-vs-3 split.
- **LOC anchor:** compute `git diff <predecessor-tag>..HEAD --stat` against the bucketed surface, print
  the number. Kills the stale-`4902cca` / conflated-scope class.
- **open gaps:** count `reports/proposals/*.proposal.md` minus shipped, surface the live set.

`STATUS.md` carries a banner: `GENERATED by tools/gen_status.py -- do not edit; re-run to refresh.`
A `<!-- generated <ISO-ts> from <sources> -->` stamp makes staleness self-evident.

### 3.3 A drift-check (the loud failure path)

A `tools/check_drift.py` (advisory pre-flight, mirrors `soak-trigger-matrix` ethos) that FAILS when a
hand-doc status claim contradicts the generated source -- e.g. a doc says "P5 0%" while the gauge derives
95%, or a ship-gate anchor SHA != the predecessor tag. Default-loud: a contradiction is a non-zero exit,
not a silent stale number. This is the same "make leakage a loud failure path" principle the polarity
rule already uses (CLAUDE.md).

## 4. Why this is the lightest thing that works

- **No new prose to maintain.** `STATUS.md` is output, not input. Ownership stamps are write-once.
- **Drift becomes impossible-by-construction for the numbers** (generated) and **loud for the prose**
  (drift-check), instead of silent.
- **Reuses existing signals** (git tags, gov.db, the ledger, ADR front-matter) -- no new source of truth,
  it INDEXES the truths that already exist.
- It directly answers the ask: the value the operator wanted from "a single SOT" is *trustworthy single-
  glance current state*; generation + drift-check deliver that without the maintenance trap a hand-doc is.

## 5. Compliance

- **Firewall (G1):** PASS -- pure SM docs/tooling; no certPortal coupling.
- **Polarity (G2):** PASS -- if the gauge reads gov.db it uses the standard `project_slug NOT IN
  (STREAM_MANAGER_PROJECT_SLUGS)` self-exclusion; generated, read-only.
- **ADR-18 MUST floor:** PASS -- docs + a new read-only `tools/gen_status.py` + `tools/check_drift.py`;
  touches no FROZEN runtime surface. New tooling under `tools/**` is Tier-1 soak per the matrix.
- **Frozen-surface note:** none touched.
- **New-envelope note:** none (no bus envelope added).

## 6. Recommended sequencing (if accepted)

1. Stamp the ownership table (section 2) into a short `docs/CANONICAL-OWNERS.md` + front-matter on the
   top ~10 docs. (S, docs-only.)
2. `tools/gen_status.py` -> `STATUS.md` generated from git + gov.db + ledger. (M, read-only tool.)
3. `tools/check_drift.py` advisory pre-flight + wire into the `/goal` self-check or ship-gate. (M.)
4. Retire / mark-historical `docs/v10-mvp-status.md` once the gauge is generated. (S.)
