What this mockup demonstrates (review before any code)

  • Mounts in place of #18: flip the flag OFF and the slot falls back to the single-action #18 ConfidenceChip stand-in -- the two co-pilots are never shown together.
  • Paired signals (ADR-18 M4/M5): every action carries a literal VERB + a tabular PERCENT + a confidence WORD (high / moderate / low) + a BINDING / ADVISORY text tag. Strip all color and the meaning still reads.
  • Only APPROVE is binding: it routes the host commit (logged as "commit-approve"). TUNE pre-stages the OVERRIDE picker, ESCALATE focuses the escalation control, SNOOZE dims the row client-side -- none mutate the server in v1.
  • Low confidence is dimmed AND labelled: the SNOOZE row (< 60%) is dimmed and carries a literal "low confidence" word -- the dimming emphasises the label, it is never the sole cue, and a dimmed macro never auto-fires.
  • Keyboard (M17): click the palette (or Tab to it), then Up / Down move the highlight, Enter fires the highlighted macro, Ctrl/Cmd+Enter fires APPROVE directly, Esc collapses and focuses Override (dissent one key away).
  • State never implicit: the SAMPLE DATA / FALLBACK RANKING tag shows the ranking provenance; switch it with the "Ranker source" control above.
  • Domain-agnostic: every label and rationale is generic governance taxonomy. No monitored-project vocabulary; the palette renders only on rows the dock already self-excludes upstream (G2 polarity).
