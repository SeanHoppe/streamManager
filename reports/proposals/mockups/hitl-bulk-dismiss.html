What this mockup shows

  • Glance-or-click path: press Alt+D (or click the gear) -- the modal opens, on-demand fetches the polarity-safe triage list, and pre-applies the default older-than-N & below-confidence preset so the right rows are already checked.
  • Default sweep: conf < 0.55 AND age > 120s. In the demo set that auto-checks 4101 + 4103 (old + low-confidence); 4102 is below confidence but younger than 120s, and 4104 is above 0.55 -- so the slider and age field visibly damp false positives.
  • Paired label+color (M4): every row carries two badges -- a trigger badge (WARN / OBSERVING) and a proposed-action badge (BLOCKED / WARN / OBSERVING) -- each a colored dot plus an uppercase literal label, so a colorblind operator reads it from text alone.
  • Never an implicit select-all: every checkbox reflects the preset and is individually overridable; unchecked rows dim and strike so you can see exactly what survives.
  • Absolute HITL gate: CONFIRM is the lone explicit commit and loops the EXISTING POST /api/hitl/resolve (resolution 'dismissed') once per checked row. Esc / Cancel close WITHOUT dismissing -- nothing auto-resolves.
  • Keyboard trap: focus lands on the first control; Tab / Shift-Tab cycle inside the trap; a polite aria-live region announces "Dismissed N rows. M remain."
  • Beta-gated: the gear, the Alt+D binding, and any fetch are absent until the flag is ON -- OFF is a pure no-op.
