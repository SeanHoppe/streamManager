BETA feature #5 -- constrained v1

Breach Cartography: temporal decision causation

When the one focus-stealing alarm fires -- a governance negative regression -- this turns "something broke" into a one-click causal decision map plus a ranked surgical-revert pick, so the operator reverts the offending verdict instead of blindly rolling back the whole run.

Build gated

UI is approvable now, but the BUILD is gated on an ADR-18 amendment. The proposal as filed touches two FROZEN surfaces (message_bus envelope schema + governance.py decision-flow hooks via a maturity_snapshots table). v1 CONSTRAINED ships WITHOUT those: no new table, no governance.py hook; per-decision maturity_delta is unavailable as live data and is derived ONLY from the coarse RingDelta already on the alert event (governance.py:710-713), or shown in mock. Endpoints are additive-only (GET /api/breach/cartography reads decisions/messages/patterns; POST revert-accept reuses the existing /api/hitl/annotate path).
