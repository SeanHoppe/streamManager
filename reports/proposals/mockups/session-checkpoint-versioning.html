Session checkpoint versioning is OFF. When OFF, the feature is a true no-op: the lane "ckpt" affordances above do not mount, the drawer registers no poller and issues no fetch, and no /api/sessions/{id}/checkpoints read happens. The lanes are exactly the shipped Frame A. Enable it in Settings > BETA features.
Where it lives

NOT a fourth frame -- the three frames stay present and calm (M1). A right-side DRAWER (the SettingsDrawer side-tray idiom) opens from the bespoke "ckpt N" affordance on each governed lane header.

Two clicks to a story

One click "Checkpoint now" writes a digest snapshot (<100ms INSERT) -- it never rewinds the live session. Arm two timeline nodes -> Compare renders a pre-computed delta manifest (<500ms). Zero network on the verdict hot path.

Color is never alone (M4)

Every delta = TEXT label + signed number + a reinforcing tone dot. Confidence drift rides a "v" / "^" / "=" glyph + signed number, never hue. A zero escalation delta collapses to a dimmed dash. Passes axe 0-serious.

SM-self is never eligible (G2)

The affordance is absent on any SM-slug lane -- polarity is structural, not a disabled state. A POST against an SM-self id returns:

HTTP 400 -- written:false
{"detail":"refusing to checkpoint SM-self session","written":false} and getCheckpoints returns checkpoints:[].

UI approvable -- build adds a new additive table. Footprint: new-table (session_checkpoints + checkpoint_diffs in gov.db), three additive read/write endpoints (POST /api/sessions/{id}/checkpoint, GET .../checkpoints, GET .../compare), and four new gated Svelte components. No FROZEN surface is touched; the sessions table is NOT frozen (additive-ALTER precedent). G1 / G2 / ADR-18 PASS. Constraints carried from the vetted brief into the build:
  • Self-refusal status is HTTP 400 (per the _reject_sm_own contract), corrected from the proposal's 403.
  • The mock's own_session_id field must be documented in the endpoint spec OR removed from the mock before build.
  • Problem statement stays domain-agnostic -- generic long-running non-SM session example, no monitored-project vocabulary (G1).
  • New bus envelope: checkpoints are write-to-DB only (no envelope assumed). If the build emits one, same PR adds cassette_record.py + soak_driver.py coverage.
