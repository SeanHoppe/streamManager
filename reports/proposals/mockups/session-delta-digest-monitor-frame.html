The glance (return-to-desk, ~3 seconds)

You were heads-down resolving a HITL row in beta-pipeline for two minutes. Sessions alpha-service, gamma-batch and others streamed decisions the whole time. The SessionRail tells you the live STATE ("alpha-service: ACTIVE 3") -- but not whether those 3 are brand-new since you last cared about alpha, or were already there. This panel answers exactly that: rows with a non-zero delta float to the top with an amber left-edge AND a paired text badge; a caught-up lane reads a literal [no change] and stays calm slate.

The acknowledgment is the inbox "mark read" gesture: clicking a row selects that session (routed through the rail's guarded selectSession, G2-safe) AND writes that lane's last_seen watermark to localStorage, so its delta resets to [no change]. Try it in the rail -- click a changed row, watch it drop to calm and the MOVED count fall.

Paired label + color badge (M5 -- color is never the only channel)

+4 dec amber -- new decisions in the window (slate +0 dec at zero)
GUIDE 2 INTERVENE 1 BLOCK 1 per-band sub-chips, each carrying the band WORD
2 esc red border -- escalations DERIVED from decision bands (no escalation table; server.py:2303-2307)
1 L4 green -- patterns crossing into level 4 (patterns.level, server.py:2018)
no change slate dashed -- a single calm word-chip; never an empty colored box, never a bare amber glow

Remove ALL hue and every row still reads as a full delta: the text+count is the durable signal; the amber left-edge is decorative emphasis only. Each row's aria-label restates the whole delta in words, so a screen-reader never depends on color.

Calm + safety contract

  • Additive RAIL CHROME only -- NO 4th frame (M1/M6). The three frames are untouched; the panel is collapsed-by-default and auto-expands ONCE on return-to-tab (Page Visibility) when any lane moved.
  • Read-only GET /api/session-delta -- no new bus envelope, no frozen surface touched. last_seen is a CLIENT watermark in localStorage (key sm.next.session-delta-seen, per session_id) passed back as since_ts; no new table, no sessions-schema write.
  • Polarity G2: the endpoint excludes SM-self (project_slug NOT IN {streamManager} AND session_id != self); the panel shows the on-screen excluded_self readout in the footer. No SM-own row is ever constructed.
  • Identity is rendered FROM DATA (project_slug) -- generic alpha-/beta-/gamma- placeholders only, no monitored-project vocabulary (M8/M16).
  • Flag OFF => the whole panel renders nothing and registers no poller (no-op gate). Flip the switch above to see it collapse to the gated-absence line.

Keyboard path (M17 / WCAG AAA)

  • The panel header is a real toggle <button> (aria-expanded / aria-controls). Enter / Space collapses + expands the panel.
  • Each row is in tab order; Enter / Space selects + acknowledges (writes the watermark, resets the lane to [no change]). The "mark all caught up" button is focus-reachable.
  • Esc collapses the panel and returns focus to the rail title.
  • An aria-live="polite" region announces on auto-expand ("3 sessions changed: alpha-service +4 decisions 2 escalations; gamma-batch +1 decision"). The 2px amber focus ring (--accent) with 2px offset matches the global ring.
