The glance + the one click (mid-afternoon, three quiet lanes)

Three background sessions are humming; the dashboard is ambient glass on your second monitor. One that was mid-refactor goes quiet. WITHOUT this, you notice the quiet by chance, then reconstruct it across four panes: /api/sessions (last move), /api/decisions (was the last verdict a BLOCK?), /api/lifecycle/jobs (still a job open?), /api/hitl/pending (waiting on YOU?). Four panes, a mental diff, a guess -- the constant tax of not being able to trust the calm.

WITH it ON, the rail RE-SORTS and the sick lane jumps to the top wearing its literal word. GLANCE: your eye lands on the top lane, reads STALLED (or BLOCKED) -- no scroll. CLICK: on the STALLED lane, one "Worth waiting?" button fires the async classifier; a beat later the lane resolves to a paired LIVE ("leave it -- work in progress") or DEAD ("archive or move focus") chip. The 7-step UC-01 collapses to <30s.

Try it: the top STALLED lane resolves LIVE, the SLOW lane below it resolves DEAD, and one is wired to the error-degrade path -- click each "Worth waiting?".

5-state paired label + color badge (M4 -- color is never the only channel)

BLOCKED red, 2px -- governance_negative_regression in the last 2min
ATTENTION 2 amber -- pending_hitl_count > 0 (count inline)
SLOW amber-dim -- GUIDE/INTERVENE >50% rolling 10min
STALLED slate-cool (NEW token) -- no message >5min. Absence-of-signal, not danger.
HEALTHY green -- active, no open actions

The "Worth waiting?" result chip is itself paired:

LIVE green -- open job + recent command; worth waiting
DEAD slate (NOT red) -- terminal state, no activity. Dead is a fact, not an error.
UNAVAILABLE violet -- classifier failed; "check manually". Never silently reads as live/dead.

Why position is the third, color-free channel

The urgency SORT is the hierarchy: the worst health_signal is simply highest, so a colorblind operator gets the full signal from ORDER + WORD alone -- color removed entirely and the UI still works (the strongest possible M5 compliance). The rail stays calm slate; the ONLY saturated surface is the single top lane when something is BLOCKED. No card grid, no dashboard-of-meters.

Calm + safety contract

  • The classifier is strictly OPERATOR-triggered and read-only-advisory -- it NEVER auto-foregrounds or pulses (M2). It spends a Sonnet call only when the human clicks. The lone permitted foreground stays reserved for the real escalation stream.
  • The verdict resolves IN-PLACE in the lane (no modal, no focus steal) -- monitor-first.
  • Read-only endpoints: GET /digest, GET /sessions/digest, POST + GET /health-check. NO new bus envelope, NO new table, NO governance-flow mutation. The async classifier uses the existing cli_pool public send seam without modifying it (frozenTouch=false, pending the amendment).
  • G2 polarity: the server excludes SM-self by project_slug + session_id; excluded_self is surfaced in the payload and the rail footer. No SM-own lane is ever constructed.
  • M16: every lane identity is rendered FROM DATA (project_slug) -- placeholder names only, no monitored-project vocabulary.
  • Flag OFF => zero pollers, the rail falls back to its default started-at order and #32's badge (if #32 is on). Flip the switch above to see it collapse.

Keyboard path (M17)

  • Each lane is a real <button> -- Tab to it, Enter / Space selects.
  • The "Worth waiting?" control is a SEPARATE focusable <button> in the same lane. Tab order: lane -> classify -> expand.
  • 2px amber focus ring offset 2px on every interactive element, matching the rail.
  • Reduced motion: the classifying spinner degrades to a static "classifying..." text; no lane transition animates.
