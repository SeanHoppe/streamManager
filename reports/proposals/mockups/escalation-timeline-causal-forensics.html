Build gated This UI is approvable now, but the BUILD is gated on an ADR-18 amendment authorizing two additive-only tables on the FROZEN message_bus.py schema: escalations(escalation_id, decision_id, event_type, agent_id, triggered_at, message_id, proposed_action, confidence, session_id, dismissed_at) and escalation_dismissals(decision_id, dismissed_at). Both are additive-only (Rule 1); no new bus envelope kind. Precedent: the schema layer already permits metadata-only extensions. Approve the mockup; the build proceeds post-amendment.

Footprint. new-table (frozenTouch -- needs amendment), no new envelope. Two new read endpoints GET /api/escalations?session_id&limit and GET /api/escalations/{decision_id}/context?window_ms, one write endpoint POST /api/escalations/{id}/dismiss (writes escalation_dismissals only -- never decisions/escalations, off the verdict hot path, M18). Two new components EscalationTimeline.svelte + DecisionDiff.svelte, registry key escalation-timeline-causal-forensics (group Monitor, default OFF). Agent attribution is persisted at decision-write time via MessageBus.record_escalation(decision_id, event_type, agent_id, ...) -- no expensive time-window JOINs at read. Event-type classification reuses escalation.js + sse.js escalationForDecision -- no new decisions column. Reuses the existing action tokens --c-allow / --c-guide / --c-intervene / --c-block -- no new color tokens. Every state pairs a literal action/event WORD with its color (ADR-18 M4/M5); severity is expressed as TYPOGRAPHIC weight, not chrome. axe target: 0 serious, color-contrast PASS, AAA text targets on the obsidian/phosphor/paper card surfaces.
