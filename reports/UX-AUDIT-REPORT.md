## Soak / non-SM-session GAPS (#2) (2)

### [HIGH] Learn Mode pattern contamination: soak synthetic pairs write to unfiltered table

- **Claim:** The soak driver's _run_lm_dialogue_pump() function writes synthetic Learn Mode dialogue pairs to learn_patterns_canonical without project_slug isolation, contaminating the global pattern table that bias_for() reads during governance verdicts for all sessions (including non-SM sessions).
- **Value argument:** Soak synthetic patterns should be isolated to soak-only sessions. Writing them to the shared learn_patterns_canonical table means every future governance decision reads contaminated bias hints that were trained on synthetic SM behavior, not on real user/operator decisions from non-SM projects. This creates an attack surface where soak performance improvements could accidentally optimize for SM's own patterns rather than cross-project generalization.
- **Rationale:** The finding identifies a structural isolation failure. The learn_patterns_canonical table lacks a project_slug column (message_bus.py:199-209), consolidate_patterns() writes with no session/project filtering (decay.py:160-167), and bias_for() reads without any WHERE clause restriction by project origin (learn_categorizer.py:846-854). While soak sessions explicitly use project_slug="soak" (soak_driver.py:1747), this isolation is lost the moment synthetic patterns are written to the unfiltered canonical table. Every subsequent governance verdict reads this contaminated pool via bias_for(content, self.bus) (governance.py:897) without the available session_id parameter. This violates POLARITY (SM should monitor NON-SM sessions only) and creates an attack surface where soak-optimized patterns bias all future verdicts.
- **Grounding:** tools/soak_driver.py:1298-1301; tools/soak_driver.py:1747; src/stream_manager/decay.py:160-167; src/stream_manager/message_bus.py:199-209; src/stream_manager/learn_categorizer.py:790-795; src/stream_manager/learn_categorizer.py:846-854; src/stream_manager/governance.py:256; src/stream_manager/governance.py:897
- **Refute:** Refuters confirmed the gap: learn_patterns_canonical lacks project_slug schema support, consolidate_patterns writes without session context, bias_for reads without filtering, and governance calls bias_for without passing available session_id.
